# PRD：BossSay - Boss 直聘 AI 智能打招呼助手

## 一、产品概述

| 维度 | 内容 |
|------|------|
| **产品名称** | BossSay - Boss 直聘 AI 打招呼助手 |
| **产品形态** | Chrome/Edge 浏览器扩展 (Manifest V3) |
| **一句话定位** | 基于 AI 的 Boss 直聘智能打招呼插件，让你的每一条消息都精准、有诚意 |
| **核心价值** | 读取岗位 JD + 结合你的简历/经历/作品 → 生成个性化打招呼消息 → 提高 HR 回复率 |
| **目标用户** | 在 Boss 直聘上求职的互联网从业者（尤其是技术岗） |
| **开源协议** | MIT License |

## 二、核心痛点

| 痛点 | 现状 | 解决方案 |
|------|------|----------|
| 千篇一律的打招呼 | "你好，我对这个岗位很感兴趣" → HR 已读不回 | AI 结合 JD 和你的经历生成针对性消息 |
| 不知道怎么展示自己 | 有实力但不会表达 | AI 自动匹配经历与 JD 要求，提炼亮点 |
| 海量投递效率低 | 每个岗位手动写消息，耗时耗力 | 一键生成，秒出结果 |
| 找不到合适的切入点 | 不知道自己的哪些经历跟这个岗位最相关 | AI 分析 JD 关键词，自动匹配最相关经历 |

## 三、功能设计

### 3.1 用户数据录入（插件 Popup 页面）

用户在插件的设置页面中录入以下信息，所有数据存储在 `chrome.storage.local`：

| 数据项 | 说明 | 必填 |
|--------|------|------|
| **简历文本** | 粘贴简历全文或关键段落 | ✅ |
| **个人经历** | 项目经历、工作经历（自由文本） | ✅ |
| **技能标签** | 技术栈、专业技能 | ✅ |
| **GitHub 仓库** | GitHub 用户名或重点仓库链接 | ❌ |
| **作品集链接** | 个人网站、作品集 URL | ❌ |
| **自我介绍** | 一段通用的自我介绍（作为 AI 的基础素材） | ❌ |
| **API 配置** | API 端点、API Key、模型名称 | ✅ |
| **消息风格偏好** | 专业正式 / 热情亲切 / 幽默轻松 / 自定义 | ✅ |
| **发送模式** | 自动发送 / 手动确认 | ✅ |

### 3.2 核心功能：智能生成打招呼消息

**触发方式**：在 Boss 直聘岗位详情页，点击浏览器工具栏的插件图标 / 页面内注入的按钮

**处理流程**：

```
用户打开岗位详情页
    ↓
Content Script 注入页面
    ↓
提取 JD 内容 (职位名/要求/薪资/地点等)
    ↓
构造 Prompt (JD + 简历 + 经历 + 风格)
    ↓
调用 AI API (用户自选模型)
    ↓
AI 生成消息
    ↓
消息展示/编辑 (可修改后发送)
    ↓
填入输入框 (自动/手动发送)
```

### 3.3 消息风格选项

| 风格 | Prompt 策略 | 适合场景 |
|------|-------------|----------|
| **专业正式** | 突出技能匹配、项目经验、技术深度 | 技术岗、大厂 |
| **热情亲切** | 突出对岗位/公司的兴趣、个人热情 | 创业公司、小团队 |
| **幽默轻松** | 用轻松的语气展示实力，有记忆点 | 有个性的团队、创意岗位 |
| **自定义** | 用户自定义 system prompt | 高级用户 |

### 3.4 AI Prompt 模板（核心）

```
System Prompt:
你是一个专业的求职顾问。你的任务是根据求职者的背景和目标岗位的JD，生成一条精准、有诚意的Boss直聘打招呼消息。

规则：
1. 消息长度控制在 100-200 字
2. 不要使用"我对这个岗位很感兴趣"等空洞表达
3. 必须提及至少一个与 JD 要求匹配的具体经历/技能
4. 语气{风格要求}
5. 不要用"您"，用"你"，保持平等对话感
6. 最后可以适当提一个跟岗位相关的问题，展示你的思考

求职者背景：
{简历文本}
{个人经历}
{技能标签}
{GitHub/作品集}

目标岗位：
职位名称：{职位名}
岗位要求：{JD 内容}
薪资范围：{薪资}
工作地点：{地点}
```

## 四、技术架构

### 4.1 技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| 扩展框架 | Manifest V3 | Chrome/Edge 兼容 |
| 前端 UI | HTML + CSS + Vanilla JS | 轻量，无框架依赖 |
| 数据存储 | chrome.storage.local | 本地存储，隐私安全 |
| AI 接口 | OpenAI 兼容 API | 支持通义千问、文心等国产模型 |
| 构建工具 | 无（纯原生） | 降低开源门槛 |

### 4.2 文件结构

```
boss-say/
├── manifest.json           # 插件配置
├── popup/                  # 弹出页面（设置 + 生成）
│   ├── popup.html
│   ├── popup.css
│   └── popup.js
├── content/                # 内容脚本（页面注入）
│   └── content.js          # 提取 JD、注入按钮、填入消息
├── background/             # 后台服务
│   └── service-worker.js   # API 调用、消息处理
├── options/                # 设置页面
│   ├── options.html
│   ├── options.css
│   └── options.js
├── lib/                    # 工具库
│   ├── ai-client.js        # AI API 封装
│   ├── storage.js          # 存储封装
│   └── parser.js           # JD 解析
├── icons/                  # 图标
│   ├── icon16.png
│   ├── icon48.png
│   └── icon128.png
└── README.md               # 项目文档
```

### 4.3 API 兼容性设计

支持所有 OpenAI 兼容 API 的国产模型：

| 模型 | API 端点 | 模型名称 |
|------|----------|----------|
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-turbo` / `qwen-plus` |
| 文心一言 | `https://aip.baidubce.com/rpc/2.0/ai_custom/v1/wenxinworkshop` | `ernie-speed-128k` |
| 智谱 AI | `https://open.bigmodel.cn/api/paas/v4` | `glm-4-flash` |
| Moonshot | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |

用户只需配置：`API Base URL` + `API Key` + `Model Name`

### 4.4 Boss 直聘页面解析策略

Boss 直聘岗位详情页的关键 DOM 结构：

```javascript
const SELECTORS = {
  jobTitle: '.job-name',
  salary: '.salary',
  location: '.job-area-wrapper',
  jdContent: '.job-detail-section',
  requirements: '.job-requirements',
  companyInfo: '.company-info',
  bossName: '.info-primary .name',
};
```

> 注意：Boss 直聘的 DOM 结构可能更新，需要提供选择器配置让用户自行调整

## 五、用户流程

### 5.1 首次使用流程

```
安装插件 → 点击图标 → 引导页 → 填写简历/经历 → 配置 AI API → 开始使用
```

### 5.2 日常使用流程

```
打开 Boss 直聘岗位详情页
    ↓
点击插件图标 / 页面内按钮
    ↓
插件自动提取当前页面 JD
    ↓
选择消息风格
    ↓
点击"生成消息"
    ↓
AI 生成个性化消息（显示在弹窗中）
    ↓
[手动模式] 编辑消息 → 点击"填入输入框" → 确认发送
[自动模式] 直接填入输入框并发送
```

## 七、竞品分析

| 产品 | 优势 | 劣势 |
|------|------|------|
| Boss 直聘自带模板 | 官方功能，使用门槛低 | 模板千篇一律，无个性化 |
| ChatGPT 手动 | 效果好 | 需要手动复制 JD、切换页面，效率低 |
| **BossSay** | **一键生成、针对性强、支持自定义模型** | 需要配置 API |

## 八、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Boss 直聘 DOM 结构更新 | JD 提取失败 | 提供选择器自定义配置，快速迭代修复 |
| AI 生成消息质量不稳定 | 用户体验差 | 提供消息重新生成、手动编辑功能 |
| API 调用成本 | 用户经济负担 | 推荐使用免费/低价模型（如通义千问免费额度） |
| 被 Boss 直聘检测为自动化工具 | 账号风险 | 不做自动发送，模拟人工操作间隔 |

## 它能做什么

在 Boss直聘搜索页，一键扫描岗位信息，AI Agent 自动分析 JD、匹配简历、生成个性化打招呼消息。

**传统方式：**

> "你好，我对这个岗位很感兴趣，希望有机会进一步沟通。"

**BossSay 生成：**

> "我有 2 年 React 开发经验，独立负责过日活 10 万的小程序重构。看到你们要前端开发，技术栈很匹配。随时到岗，可以长期实习，希望转正。请问团队目前主要用 React 还是 Vue？"

HR 看到第二条，回复率高得多。

## AI 架构

BossSay 不是简单的"调一次 API"。它实现了一个 **3 步推理链的 Agent Loop**：

```
Step 1: 分析 JD + 匹配简历
  → AI 提取岗位核心要求、关键技能
  → 自动匹配简历中的相关经历和技能
  → 计算匹配度分数

Step 2: 生成消息
  → 基于匹配分析决定消息策略（高匹配→展示技能，低匹配→展示潜力）
  → 结合求职者到岗时间、实习时长、转正意愿
  → 按三段式结构生成初稿
  → 自我审查：检查是否编造、是否超长、是否有空洞表达

Step 3: 修正（如果审查发现问题）
  → 根据审查意见自动修正消息
```

每一步都有结构化的输入输出，形成一个**可视化的推理链**。用户可以在弹窗里展开查看 AI 的完整思考过程。

### 评估反馈闭环

```
生成消息 → 记录匹配度/风格/推理链
  → 用户标记"已发送"
  → 用户标记"HR 是否回复"
  → 按风格统计回复率
  → 自动推荐最优风格
```

## 快速开始

### 1. 安装

1. 下载本仓库（Code → Download ZIP）并解压
2. 打开 Chrome/Edge，地址栏输入 `chrome://extensions/` 或 `edge://extensions/`
3. 打开右上角「开发者模式」
4. 点「加载已解压的扩展程序」，选择解压后的文件夹

### 2. 配置 AI

1. 点击工具栏的 BossSay 图标
2. 切换到 **⚙️ 设置** 页签
3. 填写 API 配置：

| 提供商 | API 地址 | 推荐模型 |
|--------|----------|----------|
| DeepSeek | `https://api.deepseek.com/v1` | deepseek-chat |
| OpenAI | `https://api.openai.com/v1` | gpt-4o-mini |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | qwen-plus |

4. 点「💾 保存」→「🔗 测试连接」

### 3. 填写简历

切换到 **👤 资料** 页签：

- **上传 PDF** — AI 自动提取简历信息（支持文字版和扫描件 OCR）
- **手动填写** — 简历摘要、工作经历、技能标签等
- **求职状态** — 到岗时间、实习时长、求职类型、转正意愿

### 4. 使用

1. 打开 Boss直聘搜索页（`zhipin.com/geek/jobs`）
2. 点击页面右下角的 **BossSay** 按钮
3. 点 **🔍 扫描当前页面岗位** → 自动填入职位/公司/薪资/地点
4. （可选）手动修正任何字段，或粘贴 JD
5. 选择消息风格，点 **✨ AI 生成打招呼消息**
6. 查看匹配度和推理链，确认后点 **📝 填入输入框**

### 关键技术决策

| 决策 | 原因 |
|------|------|
| Agent Loop 而非单次 API | 多步推理更可控，每步有结构化输出，便于调试和优化 |
| 3 步而非 5-6 步 | 平衡推理深度和响应速度（3 次 API 调用，6-15 秒） |
| 全字段可编辑 | Boss直聘反爬随时变化，自动提取+人工校验是最稳方案 |
| Service Worker 代理 API | 避免 CORS，统一 API 调用路径 |
| PDF → 图片 → AI OCR | 不依赖额外 OCR 库，用视觉模型识别扫描件 |
| 评估反馈闭环 | 数据驱动优化，追踪哪些风格/策略回复率高 |

## 支持的模型

任何兼容 OpenAI Chat Completions API 的模型均可使用。

Agent Loop 每次生成需要 3 次 API 调用，推荐使用响应快的模型（如 deepseek-chat、gpt-4o-mini）。

扫描件 PDF 的 OCR 功能需要支持图片输入的视觉模型（如 GPT-4o、Claude Sonnet）。
